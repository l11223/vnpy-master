# GitHub Actions 工作流说明

## 📋 当前有两个工作流

### 1. `pythonapp.yml` - VNPY原版工作流

**用途**：
- Windows系统上的代码检查（Lint）
- 类型检查（mypy）
- 构建Python包

**触发条件**：每次push都会触发（`on: [push]`）

**运行环境**：`windows-latest`

**是否 needed**：❌ **不需要**（这是VNPY原版的，我们只做Mac打包）

### 2. `vnpy_mac_dmg.yml` - Mac DMG打包工作流（我们创建的）

**用途**：
- Mac系统上的DMG自动打包
- 生成可安装的.dmg文件

**触发条件**：
- 推送到main分支自动触发
- 手动触发（workflow_dispatch）

**运行环境**：`macos-12`

**是否 needed**：✅ **需要**（这是我们需要的）

## 🔧 解决方案

### 方案1：禁用pythonapp.yml（推荐）

因为我们只需要Mac打包，不需要Windows的代码检查。

**操作**：
1. 重命名或删除 `pythonapp.yml`
2. 或者修改触发条件，让它不自动运行

### 方案2：保留但修改触发条件

如果以后可能需要，可以修改触发条件，让它只在特定情况下运行。

## 💡 我的建议

**推荐：禁用pythonapp.yml**

原因：
- ✅ 我们只需要Mac打包
- ✅ 减少不必要的运行（节省GitHub Actions配额）
- ✅ 避免混淆
