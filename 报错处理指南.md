# GitHub Actions 报错处理指南

## 🚨 如果报错了，怎么办？

### 方式1：直接复制错误信息给我（推荐 ⭐）

**最简单、最快速、最准确！**

1. 在GitHub Actions页面，点击失败的步骤（红色❌）
2. 滚动到错误信息部分
3. **全选并复制**错误日志（Ctrl+A, Ctrl+C 或 Cmd+A, Cmd+C）
4. 直接粘贴给我

**优点**：
- ✅ 我能看到完整的错误信息
- ✅ 能立即定位问题
- ✅ 修复最快

**示例**：
```
你：报错了！
我：把错误信息发给我
你：[粘贴错误日志]
我：[立即分析并修复]
```

### 方式2：告诉我报错了，我去看

如果你不方便复制，也可以：

1. 告诉我："报错了，在XXX步骤"
2. 提供运行记录的链接或编号
3. 我会指导你如何查看，或者你截图给我

**优点**：
- ✅ 不需要复制大量文本
- ✅ 我可以指导你查看

**缺点**：
- ⚠️ 我无法直接访问GitHub（除非配置MCP）
- ⚠️ 需要你提供链接或截图

### 方式3：安装GitHub MCP（可选）

如果你想让我直接查看GitHub Actions日志，可以配置GitHub MCP。

**优点**：
- ✅ 我可以直接查看GitHub信息
- ✅ 更自动化

**缺点**：
- ⚠️ 需要配置（需要GitHub Token）
- ⚠️ 不是必需的，方式1已经足够

## 📋 推荐流程

### 最佳实践（推荐）

```
1. 看到报错 → 点击失败的步骤
2. 复制错误日志（全选复制）
3. 发给我："报错了，错误信息如下："
4. [粘贴错误日志]
5. 我立即分析并修复
```

### 快速复制技巧

**Mac**：
- 全选：`Cmd + A`
- 复制：`Cmd + C`
- 或者：右键 → Select All → Copy

**Windows**：
- 全选：`Ctrl + A`
- 复制：`Ctrl + C`

## 🔍 需要复制的关键信息

### 最小信息（必须）

1. **错误类型**：ModuleNotFoundError, ImportError, FileNotFoundError等
2. **错误信息**：完整的错误堆栈
3. **失败步骤**：哪个步骤失败了

### 完整信息（更好）

1. 失败的步骤名称
2. 完整的错误日志（从错误开始到结束）
3. 如果有多个错误，都复制
